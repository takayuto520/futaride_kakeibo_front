# デザインガイドライン

## 1. 目的と適用範囲

このガイドラインは、ふたりで家計簿の UI/UX 品質を一貫して保つための基準です。

- 対象: Webアプリの全画面・全コンポーネント
- 目的: 一貫性、可読性、操作性、アクセシビリティの向上
- 優先順位: ユーザーの理解しやすさ > 見た目の装飾性

## 2. デザイン原則

- 一貫性: 同じ意味の情報は同じ見た目・同じ操作で提供する
- 明瞭性: 重要情報は視線導線上で見つけやすくする
- 予測可能性: ユーザーの期待どおりに操作結果が返ること
- フィードバック: 操作結果を即時に視覚的に通知する
- 最小認知負荷: 不要な情報・装飾を減らし判断を容易にする

## 3. カラールール

### 3.1 基本色

- テーマカラーは「明るい緑系」、それ以外はモノトーンを基本とする
- カラーパレット（固定値）:

| トークン名       | カラーコード | 用途                    |
| ---------------- | ------------ | ----------------------- |
| `primary`        | `#34D399`    | メイン操作、強調ボタン  |
| `primary-hover`  | `#10B981`    | 主操作のホバー          |
| `primary-active` | `#059669`    | 主操作の押下/選択       |
| `background`     | `#F8FAFC`    | 全体背景                |
| `surface`        | `#FFFFFF`    | カード、モーダル背景    |
| `text-primary`   | `#111827`    | 主要テキスト            |
| `text-secondary` | `#4B5563`    | 補助テキスト            |
| `border`         | `#D1D5DB`    | 境界線                  |
| `disabled`       | `#9CA3AF`    | 非活性テキスト/アイコン |
| `error`          | `#DC2626`    | エラー、危険操作        |

### 3.2 状態色

- 状態別カラー（主操作ボタン）:

| 状態                   | カラーコード | 用途               |
| ---------------------- | ------------ | ------------------ |
| 通常 (Default)         | `#34D399`    | 主状態の基準色     |
| ホバー (Hover)         | `#10B981`    | ポインター重なり時 |
| 選択 (Active/Selected) | `#059669`    | 選択中、押下中     |
| 非活性 (Disabled)      | `#9CA3AF`    | 操作不可           |

- 状態別カラー（ナビゲーション）:

| 状態                   | カラーコード | 用途                   |
| ---------------------- | ------------ | ---------------------- |
| 通常 (Default)         | `#F0FDF4`    | 通常時のメニュー文字色 |
| ホバー (Hover)         | `#047857`    | ホバー時の背景色       |
| 選択 (Active/Selected) | `#059669`    | 選択中メニュー背景色   |
| 選択文字 (Active Text) | `#FFFFFF`    | ホバー/選択時の文字色  |
| 非活性 (Disabled)      | `#9CA3AF`    | 操作不可メニュー文字色 |

- 成功/警告/エラー:
  - 成功: `#16A34A`
  - 警告: `#D97706`
  - エラー: `#DC2626`

### 3.3 収支表示の文字色（例外ルール）

- 収支がプラスの場合: `#0EA5E9`（水色）
- 収支がマイナスの場合: `#DC2626`（赤色）
- 収支の色は他の状態色と用途を混同しない
- 収支値は色に加えて `+` / `-` 記号も付与する

### 3.4 運用ルール

- カラーコード直書きは禁止し、Tailwind設定のトークンを使用する
- 1画面内で意味の異なる色を過剰に混在させない
- 色だけで意味を伝えず、アイコン・ラベルも併用する

## 4. タイポグラフィ

- フォントは既定のUIフォントを使用し、可読性を優先する
- 見出しは階層を守る (h1 > h2 > h3)
- 本文は読みやすさを重視し、過度な太字を避ける
- 数値(金額等)は桁区切りを行い、視認性を担保する

推奨サイズ目安:

- ページタイトル: 28-32px
- セクションタイトル: 20-24px
- 本文: 14-16px
- 補助テキスト: 12-14px

## 5. レイアウトと余白

- 余白は 4px 基準のスケールで統一する
- セクション間は要素内余白より大きく設定する
- 主要コンテンツ幅は読みやすさを重視して制限する
- レスポンシブ時は横並びから縦積みへ自然に変化させる

## 6. コンポーネント設計

### 6.1 ボタン

- 主ボタン: 1画面に1つを基本
- 副ボタン: 主ボタンより視覚優先度を下げる
- 危険操作: 赤系 + 確認導線を用意
- 無効時: 視覚的に無効を示し、クリック不可にする

### 6.2 フォーム

- ラベルは必ず表示し、プレースホルダーのみ運用は禁止
- 必須項目は一貫した記法で明示する
- 入力エラーは項目近傍に具体的文言で表示する
- エラーメッセージは「何が」「どう違うか」を示す
- 成功通知は `alert` よりも画面内トーストを優先し、短時間で自動消去する

### 6.3 カード/リスト

- 情報の優先度に応じて階層化する
- 行全体クリックの有無を見た目で区別する
- 長文は折り返し/省略ルールを定義する

### 6.5 グラフ

- 円グラフはカテゴリ設定色と一致させる
- ラベルは可読性を優先し、必要に応じて小サイズ・非太字にする
- 狭いスライスではラベルを非表示にし、視認性低下を防ぐ
- ラベルはグラフ外や他スライス上への重なりを許容し、見切れを防止する

### 6.4 ナビゲーション

- 現在ページが一目でわかる選択状態を持つ
- ホバー時と選択時の見た目を明確に分離する
- アイコンは意味が直感的に伝わるものを使う

## 7. アイコンと画像

- アイコンサイズは 16/20/24px を基準に統一
- 線幅や角丸のトーンを揃える
- 装飾目的の画像は情報伝達を妨げない
- 意味を持つ画像には代替テキストを付与する

## 8. アクセシビリティ

- WCAG 2.1 AA を満たすコントラストを確保する
- キーボード操作のみで主要機能を完結できる
- フォーカスリングを消さない
- エラーは色だけでなくテキストでも伝える
- aria属性は意味がある箇所に適切に設定する

## 9. レスポンシブ対応

- モバイルファーストで設計する
- ブレークポイントごとの情報優先度を定義する
- タップ領域は十分なサイズを確保する
- 固定幅要素は小画面でのはみ出しを防ぐ

## 10. モーション

- アニメーションは補助的に使い、意味を持たせる
- 画面遷移・開閉・ロードなど重要な変化に限定する
- 過剰な演出や長時間アニメーションは避ける
- 速度は 150-300ms を目安に統一する

## 11. 文言ガイド

- ボタン文言は動詞で明確にする (例: 保存する)
- エラー文言は責めずに、次に取る行動を示す
- 専門用語は必要最小限とし、利用者視点で記述する

## 12. 実装ルール

- スタイルは Tailwind ユーティリティとテーマ拡張を優先
- 再利用可能なUIは components 配下に分離する
- 状態に依存する見た目はクラス設計を統一する
- デザイン変更時は本ガイドラインを更新する

## 13. レビュー観点チェックリスト

- 一貫した余白・文字サイズ・色が使われている
- 選択状態/ホバー状態/非活性状態が識別できる
- フォームの必須/エラー表示が明確である
- キーボード操作とフォーカス表示に問題がない
- モバイル表示で崩れや操作困難がない
- コントラストが不足していない
